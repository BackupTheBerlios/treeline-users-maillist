Looking great. Some minor gremlins encountered so far. With Basic Long Text,
selecting node numbering after node titles are created overwrites and
replaces the titles with numbering rather than prepending numbers to
existing titles. This is with the Windows installer version running on fully
patched WinXP.

When creating a new Field type from the Configure Data Types >> Field List
>> New Field dialog, if I enter illegal characters I get an error message
that reads "The following characters are not allowed:"   However, the list
of disallowed characters is missing from the error message.

Another potentially troublesome issue. The length of text in Basic Longtext
seems to be limited to 200 lines. I often need to clip very long content
from other sources when doing research so I have a complete record of source
materials that I reference in papers I write. Is that a trait of the Basic
Longtext template that I can work around by creating my own template or is
that a coded barrier in Treeline itself? I found a relevant setting yet when
working from the default template.

I haven't tried it yet, but wonder whether the latter issue might also raise
issues when opening a lengthy document in a supported file format such as
ODT. How does Treeline handle attempts to open an ODT file if a continuous
block of text without headings is longer than 200 lines? I.e., does
Treeline  generate an error message or truncate text without warning?

Again, Treeline is looking very cool. Nice work!
