marbux wrote:
> Looking great. Some minor gremlins encountered so far. With Basic Long Text,
> selecting node numbering after node titles are created overwrites and
> replaces the titles with numbering rather than prepending numbers to
> existing titles. This is with the Windows installer version running on fully
> patched WinXP.

That is "intended functionality"  ;)

When numbering for the first time, the user should enter a new field
name as the "Number Field".  If the name of an existing field is given,
the content of that field is overwritten with the numbers.

If you want the number shown before each title, you should give a new
field name (such as "Num") for the numbering, then change the Title or
Output format to "{*Num*} {*Name*}".

I've thought about allowing numbers to be prepended to an existing field
with existing data, but it causes problems when re-numbering.  The
program could replace number-like text at the beginning of the field
data, but that wouldn't always work, since numbering formats can vary
significantly, and the results would be bad with any field text that
originally started with a number ("2 dozen eggs").

> Another potentially troublesome issue. The length of text in Basic Longtext
> seems to be limited to 200 lines. I often need to clip very long content
> from other sources when doing research so I have a complete record of source
> materials that I reference in papers I write. Is that a trait of the Basic
> Longtext template that I can work around by creating my own template or is
> that a coded barrier in Treeline itself? I found a relevant setting yet when
> working from the default template.

The number of lines option is only used for setting the height of the
text edit box in the Data Editor view.  If an entry is longer than the
setting, scroll bars will appear on the text edit box.  The maximum of
200 lines for the editor height is reasonable (if not annoyingly high).

> I haven't tried it yet, but wonder whether the latter issue might also raise

> block of text without headings is longer than 200 lines? I.e., does
> Treeline  generate an error message or truncate text without warning?

I just tested it with 3000 lines of text - it works fine.
