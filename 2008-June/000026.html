Hi,

On Sonntag, 1. Juni 2008, John Stitzinger wrote:

> Being able to sync to something like a  Subversion Server with revision
> control is a dream of mine.

This shouldn't be too hard to do. At SUSE we (mis)used a heavily patched 
TreeLine for a while some years ago as a frontend to our feature tracking 
SVN. All the features were separate files in SVN and were read/written back 
by a patched implementation of TreeLines file open/save methods.

Depending on whether you want a specific backend to be addressed (e.g. 
Subversion) or just want to be able to go back in time on your personal 
desktop, I'd choose different approaches.

E.g. Mercurial, as a version control system written in Python, should be even 
easier to integrate with TreeLine than Subversion.

Back then when we used TreeLine with SVN the tricky part was merging if two 
users wrote to the same nodes. That of course is only an issue if you 
actually have more than one user or use the same backend from several 
machines without always committing all changes immediately.
