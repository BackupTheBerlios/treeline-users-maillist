I noticed drive "I:" in Tomas's error message. I presume TL was installed to
some removable or "not on this box" location. In the past I have bumped into
issues this technique exposed such as DOS "PATH" errors, implied searching
(up to the root of the "current" drive, or "\WINDOWS" instead of
"C:\WINDOWS", stuff like that), and other assumptions that weren't set up
quite right. These problems play havoc with DLL searches.

If what Doug suggests doesn't work, you might try uninstalling all TL's,
document your "PATH" variable, then do a clean install to C: and note "PATH"
changes. TL "should" work like this, and any "PATH" changes might help you
install TL to some other drive.

Hope this helps.
