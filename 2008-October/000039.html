TreeLine 1.2.0 on Windows can save the file with a filename using the
Unicode, but when trying to open the file after closing it, an error
occurs and cannot open it.

Steps to reproduce the symptom is shown below.

1. Run TreeLine.

2. Select [File]-[New] from menu bar.

3. Select "Default - No template" as template.

4. Without editing anything, select [File]-[Save As] from menu bar.

5. Specify an arbitrary directory (ex. C:\) as the location to save the
file, and specify an arbitrary filename using the Unicode (ex. テスト.trl
("テスト" means "test" in Japanese.)), and click the [Save] button.

No error occurs then, and the file can be saved normally.

6. Quit TreeLine.

7. Open the file with a Unicode filename saved a while ago by
double-click.

However, the following error occurs and cannot open the file.

> Errors occurred
> See the logfile 'E:\Program Files\TreeLine\lib\treeline.exe.log' for details

And the traceback outputted to the treeline.exe.log at this time is as
follows:

> Traceback (most recent call last):
>   File "treeline.py", line 164, in <module>
>   File "treeline.py", line 155, in main
>   File "treemainwin.pyc", line 527, in openFile
>   File "treemainwin.pyc", line 692, in autoSaveFilePath
> UnicodeDecodeError: 'ascii' codec can't decode byte 0x83 in position 3: ordinal not in range(128)

Guessing from this traceback, it seems that the internal filename code
conversion has failed.
Then, after changing the filename saved a while ago into the filename
NOT using the Unicode (ex. test.trl), when double-click it, the file was
opened successfully this time.

I installed the source code of TreeLine and read it a little.
However, as I am not familiar with Python, I was not able to find out a
cause.

Could you resolve this symptom?


Finally, my environment is shown below.
  OS       : WinXP Pro SP3 (Japanese Edition)
  TreeLine : 1.2.0 (Downloaded from BerliOS)
