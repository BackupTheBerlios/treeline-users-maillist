I'm trying to understand how to use conditional types in Treeline. What I
have in mind is to set up a "choice" field within a data type so that,
depending on which selection is made in "choice" various fields within the
data type will either be available or not. To clarify:

Data Type: "Beginning"
"Where" field (choice): just before, during, after

When "just before" is selected, the fields available in the "Beginning" data
type are:
POV_Character
Trouble_Foreshadowed
Symbol_Of_Struggle

When "during" is selected:
POV_Character
Motivation
Symbol_Of_Struggle

And when "after" is selected:
POV_Character
Commonplace_Threat
Symbol_Of_Struggle

Problem: The "Create Conditional Types" button is ghosted for the
"Beginning" data type.

I combed through the docs and looked at the sample file,
"sample_conditional_todo," and as far as I can tell, my "Beginning" data
type is set up the same way as sample_conditional_todo's TASK_DONE data
type, the one exception being that I'm using "choice" for the "Where" field
whereas the sample uses "boolean" for the "Done" field.

I read in the docs that in order to use conditional types, "Conditions can
be assigned only to a generic type and its associated derived types."
However, I can't seem to find a definition for "generic types" so I don't
know if I'm meeting this requirement.

Questions:
1) Am I misunderstanding the paradigm of conditional types?
2) If not, how does a data type need to be set up so that the "Create
Conditional Types" button is available?
3) Can someone give me a definition of "generic types?"
