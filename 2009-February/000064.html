Adam Byrtek wrote:
> I've just submitted a TreeLine port for MacPorts - a ports-based
> source distribution for Macs. It is not yet available in the
> repository, but can be found in the ticket:
> http://trac.macports.org/ticket/18626
> 
> There was only one system-dependent patch I had to make, I had to
> disable saving of window geometry on exit. The reason for this is that
> when this setting is enabled and TreeLine is run for the first time,
> it assumes default X/Y coordinates of 0x0. In this case on Mac the
> title bar is hidden beneath the OS menu bar and cannot be activated at
> all. Let me know if this is clear enough, if not I can provide a
> screenshot.
> 
> Maybe it would be wiser to leave the decision to the window manager
> instead of fixing the window to a default position when the geometry
> is unknown?

That makes sense.  I'll probably set the initial value for the position
to something like (-1, -1), which will be ignored until it's overwritten
with a real position.  I wonder if negatives (slightly offscreen) should
ever be valid positions?  The current version of TreeLine does ignore
them.

I'm not sure whether I'll do this in a bugfix release or wait for the
next unstable development series.

Doug.
