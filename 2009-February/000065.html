John Stitzinger wrote:
> Hi
> 
> I just want to bring up a old old feature request that I don't think has 
> ever been implemented but was discussed a bit.on the mailing list years 
> ago. It would be very useful to be able to drop a node after another 
> node as a sibling rather than as a child.
> I've seen other outliners use Shift to change the behavior from the 
> default "Paste as child" to "Paste After".
> The corresponding menu items "Paste Before" and "Paste After" would be 
> useful for very long lists of siblings rather than using "Move Up".
> 
> Either of the above methods would also make it easier to duplicate a 
> node (Shift + Control).  In this case I think the preference is for the 
> Control key to do a "Paste After" rather than "Paste Before".  I'd still 
> like a duplicate node shortcut though!
> 
> Currently I do a lot of  "Paste as Child" and "Unindent Node".
> 
> Thanks for the latest update.

It probably does make sense to add one of these, probably paste-after.
The shortcut could be Ctrl+Shift+V.  I'd prefer to make it obvious by
adding it as its own menu item, instead of changing the behavior based
on a modifier.  And using Ctrl as a modifer doesn't make sense, since
it's already used by the Ctrl+V shortcut.

It might be useful to use a shift modifier with drag-and-drop to do the
same thing, but it would violate the usual meaning of that modifier
(move vs. copy).
