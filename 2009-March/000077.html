Very true.  It works well to use <enter> to add a new sibling, 
particularly with the way tabs now work.  You can use tab to move the 
cursor to the fields (attributes) and then make changes. You then tab 
right back to the tree to add another node, again with <enter>.  I 
really like the way Treeline uses the same data type as the current node 
for new siblings and children (unless a default is setup).

Often though I find myself with a subtree with several nodes in it with 
perhaps a couple of levels where I've filled in some attributes.  Now I 
need another copy of this subtree structure that is almost the same. I'd 
like a quick way to duplicate the subtree and then I can make a change 
to a attribute or two. I'm currently using Treeline to create DNS Bind 
files for a couple of hundred Domain Names so creating duplicates would 
be a big help.

John
