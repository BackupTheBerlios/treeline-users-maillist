> Understood, and awareness within a process is a good thing (tm). Consider
> this scenario of opened and modified (TreeLine) files:
>  *) c:\file1.trl

>  *) l:\file2.trl (on another computer)
> 
> Crash the remote computer. What happens? Obviously L: changes are lost. But
> did the process survive? (Have we lost all changes to C: file as well?)

I don't think it matters where the file was stored.  TreeLine will give
an error if you try to save the file to a crashed remote drive, but
TreeLine won't crash.  I also don't think TreeLine would crash if the
drive containing TreeLine's excutable crashed, since it is already in
memory, but I haven't actually tested that.

I can't think of many scenarios where one combined session would lose
more work than separate sessions.
