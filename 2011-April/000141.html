Hi folks,

I'm using Treeline to keep track of my incoming and outgoing (snail)
mail, so I have a "Path" typed field that points to a scanned pdf
version of received and sent letters.

I'm planning to use this treeline file from multiple computers (by
syncing to and from a network share), but I expect problems with the
absolute paths that are in the datafile (haven't actually tried,
though).

Is there a way to make Treeline save relative paths (all linked files
are below the parent dir of the treeline document) instead of absoluate
path?

It actually seems that resolving relative paths is already supported for
"Picture" typed fields, but not for "Path" typed fields (also, when
using the filebrowser to set a "Picture" typed field, an absolute path
is still put into the field.

Would it make sense to:
 * Support relative paths in "Path" fields?
 * Add an option to "Path" and "Picture" typed fields: "Use relative
   paths: never / always / only for files below the .trl file" ?
