> I don't know if this request may be of some use for everyone but... well
> it would work for me!
> 
> 1 - I would like to configure a field type as "custom" and select an
> existing custom datatype in order to nest it in my item and avoid mixing
> different data types among one node's childs. I expect that this would
> lead to some kind of "endless recursive tree" if used improperly... Am I
> wrong?

It would be a recursive mess if you allow a custom field to be set to
another custom field.  Otherwise that shouldn't be a problem.

But I don't see the advantage of this approach.  Why not mix different
types of children?  You still have to set the actual field format, so
why not keep it simple and use the regular data type setting?

More detail on why you want this would be helpful.


> 2 - Another useful feature, always looking to field types, would be the
> ability to define a field as "repeatable": once the field has been
> filled on the editor view, another instance of the field should be
> created just below the filled one, in order to create a simple "in-line"
> list that could be rendered in the output view as a table...

But how would the output distinguish between repeatable field n and
field n+1?  And the views wouldn't know how many fields to show until
they looked at the contents of each node.  This seems ugly to code.

I'd recommend using child nodes when you have a variable number of
sub-items to display.


> 3 - Finally I end up with a question: is it possible, and eventually
> how, to write an output code for a data type in order to automatically
> generate a clickable "content summary" for all childs of a node?

You can create a summary now, but not a clickable one.  In the Output
tab of the Configure Data Types dialog (with Advanced features shown),
select "Child Reference" under the "Reference Level" and the choose the
applicable child type under "Reference Type".  Then just move the child
fields into the Output Format box.

Actually, it is possible to do a clickable list of children, but I can't
think of a way to get unique text in each link.  If you show the child
list as described above, but set the separator string (under Tools->File
Options) to something like this:
   ">child</a> <a href="#
Of course, the text in the Output box would need to include partial tags
in order for the first and last items to work.  This will get you the
links, but unfortunately with the same text in each one.

If you tell me more about your application for this, I might have more
suggestions.
